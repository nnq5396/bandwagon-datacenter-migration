## 搬瓦工换机房是怎么回事？先把概念说清楚

搬瓦工换机房，官方功能叫 **Migrate to another DC**（迁移到另一个数据中心），入口在 KiwiVM 控制面板左侧菜单。

这个功能做一件事：把你的 VPS 物理迁移到另一个机房，数据全程保留，迁移过程中 VPS 依然可操作，只有最后几秒到几分钟会短暂关机。迁完之后，IP 地址会变（这也是很多人换机房的真实目的——顺手换个 IP）。

有几个关键参数需要记住：

- **免费，不限次数**。每次迁移不另收费，想换多少次都行
- **会消耗月流量**。系统需要把整个磁盘镜像传输到新机房，消耗的流量和你的 VPS 存储容量成正比
- **IP 一定会变**。换了机房就是换了 IP，如果有绑定域名记得同步改 DNS 解析
- **切回原机房，IP 不一定还是原来那个**。原来的 IP 可能被别人分配走了，这是随机的

## 搬瓦工换机房的完整操作步骤

操作本身不复杂，但要先确认 IP 状态正常（下一节说为什么），然后按这个流程走：

1. 登录搬瓦工官网后台，进入 **Services → My Services**
2. 找到需要操作的 VPS，点击右侧齿轮图标，选择 **Open KiwiVM**，进入控制面板
3. 在 KiwiVM 左侧菜单找到 **Migrate to another DC**，点击进入
4. 右侧会显示当前机房位置，以及可以迁移的目标机房列表
5. 选择目标机房，点击 **Confirm on next step**
6. 确认信息无误后，点击 **Start Migration** 开始迁移
7. 迁移完成后会收到邮件通知，也可以在原来的页面查看 "Current location of this VPS" 确认当前机房

整个过程快则几分钟，慢则几十分钟，取决于 VPS 里存了多少数据。迁完就完了，不需要重新配置服务器，只需要把用到旧 IP 的地方换成新 IP。

## 最关键的一条：IP 被封后不能换机房

这是很多人会踩的坑。说实话，我一开始也以为 IP 被封之后可以靠换机房解决——毕竟换机房会换 IP，逻辑上说得通。

实际上行不通。

搬瓦工的官方规则是：**一旦系统检测到你的 VPS IP 在大陆被完全封锁，"Migrate to another DC" 功能会被自动禁用**。你不会看到"IP 被封"的提示，只会反复看到错误代码，比如 `734152` 或 `734104`，提示你稍后重试，但重试多少次都一样。

这条规则是为了防止大家用换机房的方式绕过 IP 封锁，属于搬瓦工明确公开的政策。

**IP 被封了怎么办？** 只能付费换 IP。搬瓦工后台支持自助付费换 IP，每次 $8.79，付款后通常 30 分钟内完成，新 IP 保证可用。换完 IP 之后，如果还想换机房，就可以正常操作了。

## FAQ

**Q：搬瓦工换机房会扣除多少流量？**

A：每次迁移消耗的流量等于 VPS 磁盘里的实际数据量。VPS 硬盘是 20 GB 但只用了 5 GB 的话，消耗的大约就是 5 GB 流量。流量用完会暂停服务，迁移前确认一下剩余流量。

**Q：换机房之后 IP 一定会变吗？**

A：到新机房会分配新 IP，是的。但如果你从 A 机房迁到 B 机房，再迁回 A，原来 A 的 IP 有可能还在（也有可能被别人占了）。这个是随机的，不可控，不建议靠这种方式"保留"原 IP。

**Q：KVM 套餐能迁移到 CN2 GIA 机房吗？**

A：不能。KVM 套餐和 CN2 GIA-E 套餐的机房系统是分开的，KVM 套餐只能在 KVM 机房范围内切换。想用 DC6、DC9 这类高端机房，需要购买 CN2 GIA-E 套餐。

**Q：换机房提示 "Region is full" 怎么办？**

A：说明目标机房暂时满了，等几小时再试，或者换个别的机房先用着，后续再迁过去。这种情况偶尔出现，不是账号问题。

**Q：IP 被封后花 $8.79 付费换了 IP，之后还能换机房吗？**

A：可以。付费换 IP 完成后，系统检测到 IP 恢复正常，换机房功能就会解锁，可以正常操作了。
